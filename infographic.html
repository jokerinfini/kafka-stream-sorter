Resource Management

Total memory ~2GB split between Kafka and app container.

Performance Bottleneck Analysis

Disk I/O dominates; Kafka/network and CPU are secondary.

Code-Level Performance Tuning

Batching, buffer sizes, and efficient parsing reduce I/O and network overhead.

Producer & Reader

  • Workers: NumCPU()*3
  • Writer Batch: 10,000; BatchBytes: 16MB
  • Reader fetch sizes up to 32MB

External Sort & Parsing

  • Spill/Merge buffers: 4MiB
  • Numeric ID parsing; string slicing without allocations

Scaling Strategy

Tier 1: Hardware

Faster SSDs/NVMe for temp files.

Tier 2: Parallelism

Partition topics; multiple sorters.

Tier 3: Distributed

Spark/Flink for cluster-wide sort/merge.
